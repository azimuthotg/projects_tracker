### 1. PDF รายงานภาพรวมงบประมาณ — เปลี่ยนเป็น Landscape
- **ปัญหา:** PDF เป็นแนวตั้ง (Portrait) ตารางมี 8 คอลัมน์ถูกบีบแคบ ชื่อโครงการถูกตัด
- **แก้ไข:** เปลี่ยน `build_document` ใช้ `landscape(A4)`, ปรับความกว้างคอลัมน์ใหม่ (ชื่อโครงการจาก 6.0 → 9.5 cm), เพิ่ม `page_width` parameter ใน `header_block()`
- **ไฟล์:** `apps/reports/views.py`, `apps/reports/pdf_utils.py`
- **Commit:** `7359e9e`

### 2. Executive Dashboard — การ์ดงบประมาณตามแผนก
- **ปัญหา:** Bar chart แสดงสัดส่วนแต่ไม่มียอดตัวเลข, ไม่รู้ว่าแต่ละแผนกมีงบรวมเท่าไหร่, ไม่มีจำนวนโครงการ/กิจกรรม
- **แก้ไข:**
  - ย้ายการ์ดออกจาก 3-col grid เป็น full-width แยกต่างหาก
  - เพิ่มตารางสรุปใต้ chart: แผนก | โครงการ | กิจกรรม | งบรวม | ใช้ไป | คงเหลือ | % (progress bar)
  - เพิ่มแถวรวมทั้งหมดที่ footer ตาราง
  - เพิ่ม `project_count`, `activity_count`, `dept_totals` ใน context
- **Commit:** `fae48a2`

### 3. แก้ยอดโครงการในตารางแผนก (14 vs 13)
- **ปัญหา:** KPI card แสดง 14 โครงการ แต่ตารางแผนกรวมได้แค่ 13 เพราะโครงการ [13] สังกัด "หัวหน้าสำนักงาน" ซึ่งชื่อไม่ขึ้นต้นด้วย "แผนก"
- **แก้ไข:** เพิ่มแถว non-แผนก departments จัดกลุ่มตาม department จริง แสดงด้วยสีเหลืองอ่อน (amber) + ไอคอน ▸
- **Commit:** `688deec`

### 4. แก้ Server Error 500 (hotfix)
- **ปัญหา:** `chart_dept` labels เรียก `.name` บน dict object แทน Django model ทำให้ AttributeError หลัง deploy
- **แก้ไข:** เพิ่ม `hasattr` check: `d['dept'].name if hasattr(d['dept'], 'name') else d['dept']['name']`
- **Commit:** `5b32f3b`

### 5. การ์ดแจ้งเตือนสำคัญ — ปรับ UX
- **ปัญหา:** แสดงแค่ 5 รายการ, ไม่มี link ไปดูรายละเอียด, "ดูทั้งหมด" link ไปหน้าผิด
- **แก้ไข:**
  - ลด slice จาก `[:5]` → `[:4]` เพื่อให้การ์ดสูงเท่ากัน
  - รวม title + description ในบรรทัดเดียว (เลย Deadline แล้ว | กิจกรรมที่เลยกำหนดยังไม่เสร็จ)
  - เปลี่ยน plain text → `<a>` link ไป activity detail พร้อมชื่อโครงการ
  - แทน "ดูทั้งหมด" link ด้วย **Modal popup** ของการ์ดนั้นๆ แสดงรายการทั้งหมด (ปิดด้วย overlay / ปุ่ม ✕ / ESC)
  - เพิ่ม `overdue_list_all`, `no_report_list_all`, `not_started_list_all` ใน context
- **Commit:** `1cf20e3`

### 6. การ์ดสรุปงบประมาณรายโครงการ
- **ปัญหา:** แสดงเฉพาะชื่อโครงการ + % ไม่มียอดเงิน ผู้บริหารไม่รู้ว่าโครงการที่ % สูงมีงบมากหรือน้อย
- **แก้ไข:**
  - เปลี่ยนชื่อการ์ดเป็น "สรุปงบประมาณรายโครงการ"
  - แสดง **ทุกโครงการ** (ไม่จำกัด top 6)
  - เรียงตาม % มากสุดก่อน, ถ้า % เท่ากัน/0% เรียงตามรหัสโครงการ
  - แต่ละแถวแสดง 2 บรรทัด: ชื่อ+% / งบ+ใช้+เหลือ
  - คลิกได้ → project detail
- **Commit:** `5f78131`

### 7. แก้ชื่อระบบทั่วทั้งระบบ
- **ปัญหา:** ชื่อระบบในหลายจุดยังเป็น "ระบบติดตามโครงการ" ซึ่งไม่ถูกต้อง
- **ชื่อจริง:** ระบบติดตามแผนงาน โครงการและงบประมาณ

  - `apps/reports/views.py` — footer PDF (2 จุด)
- **Commit:** `74827d5`

### 8. แก้จำนวนสมาชิกในหน้าจัดการแผนก
- **ปัญหา:** จำนวนสมาชิกและโครงการแสดงตัวเลขเดียวกัน เนื่องจาก Django Cartesian Product เมื่อ JOIN หลายตารางพร้อมกันโดยไม่ใช้ `distinct=True`
- **แก้ไข:** เพิ่ม `distinct=True` ใน `Count('members')` และ `Count('projects')` ทั้ง 2 view (`manage_dashboard`, `department_list`)
- **ผลลัพธ์ที่ถูกต้อง:** สมาชิกและโครงการแยกกันถูกต้อง
- **Commit:** `ca9c862`
