# บันทึกความก้าวหน้า — 14 พฤษภาคม 2569

## สรุปงานที่ทำวันนี้

งานวันนี้เป็นการ **ตรวจสอบและแก้ไขระบบ RBAC (Role-Based Access Control)** ให้สอดคล้องกันทั้งระบบ
หลังจากที่พบปัญหาในการใช้งานจริงว่า role "เจ้าหน้าที่" ไม่สามารถดำเนินการบางอย่างได้ตามสิทธิ์ที่ควรจะมี

---

### 1. เพิ่ม Role "ผู้บริหาร" (executive) — อ่านอย่างเดียวข้ามแผนก

- **เหตุผล:** ผู้บริหารต้องการดูภาพรวมทุกแผนกโดยไม่สามารถแก้ไขข้อมูลได้
- **การดำเนินการ:**
  - เพิ่ม `executive` ใน `get_viewable_projects()` ให้ return ทุกโครงการ
  - ปรับ Dashboard, Reports ให้ executive เห็นข้อมูลทุกแผนก
  - ซ่อนเมนู "งานของฉัน" ใน sidebar สำหรับ executive
  - เพิ่ม "ภาพรวมผู้บริหาร" ในเมนู sidebar
- **ไฟล์:** `apps/projects/utils.py`, `apps/dashboard/views.py`, `apps/reports/views.py`, `templates/components/sidebar.html`

---

### 2. เจ้าหน้าที่ไม่เห็นโครงการแผนกใน Timeline และรายการเบิกจ่าย

- **ปัญหา:** เจ้าหน้าที่ที่เป็นผู้รับผิดชอบกิจกรรม ไม่เห็นโครงการของแผนกใน Gantt Chart (ปฏิทินโครงการ) และรายการเบิกจ่ายแสดงเฉพาะรายการที่ตัวเองสร้าง
- **สาเหตุ:** Views ใช้ `get_projects_for_user()` (actionable) แทนที่จะใช้ `get_viewable_projects()` (viewable)
- **แก้ไข:**
  - `project_timeline` view → เปลี่ยนเป็น `get_viewable_projects()`
  - `get_expenses_for_user()` → เปลี่ยนให้ staff เห็น expense ทุกรายการในแผนกตัวเอง
- **ไฟล์:** `apps/projects/views.py`, `apps/budget/utils.py`

### 3. ระบบแบบฟอร์ม (Document Template Download)

- **เหตุผล:** เจ้าหน้าที่ต้องการดาวน์โหลดแบบฟอร์มมาตรฐาน เช่น แบบฟอร์มรายงานผลกิจกรรม, เอกสารเบิกจ่าย ซึ่งแผนกนโยบายและแผนกำหนดไว้
- **การดำเนินการ:**
  - เพิ่ม Model `DocumentTemplate` (ชื่อ, หมวดหมู่, ไฟล์, คำอธิบาย, เวอร์ชัน, สถานะ)
  - สร้างหน้าแสดงรายการแบบฟอร์ม (`/forms/`) สำหรับทุก role
  - สร้างหน้าจัดการแบบฟอร์ม (`/forms/manage/`) สำหรับ admin เท่านั้น
  - รองรับไฟล์ประเภท PDF, Word (.docx/.doc), Excel (.xlsx/.xls) ขนาดไม่เกิน 20 MB
  - ดาวน์โหลดผ่าน `FileResponse` พร้อม content-disposition (ดาวน์โหลดตามชื่อต้นฉบับ)
  - เพิ่มเมนู "แบบฟอร์ม" ใน sidebar ทุก role
- **Migration:** `apps/projects/migrations/0011_documenttemplate.py`
- **ไฟล์:** `apps/projects/models.py`, `apps/projects/views.py`, `apps/projects/urls.py`, `apps/projects/admin.py`, `templates/projects/forms_list.html`, `templates/projects/forms_manage.html`, `templates/projects/forms_edit.html`

### 4. เจ้าหน้าที่กด "เริ่มดำเนินการ" แล้วได้รับ 403 Forbidden

- **ปัญหา:** เจ้าหน้าที่ที่เป็น **ผู้รับผิดชอบร่วมในกิจกรรม** (responsible_persons) มองเห็นปุ่มแต่กดแล้วได้ 403
- **สาเหตุ:** View `activity_status_change` ใช้ `@role_required(['planner', 'head', 'admin'])` ซึ่งปิดกั้น staff ทั้งหมด
- **แก้ไข:** เปลี่ยนเป็น `@login_required` และตรวจสอบสิทธิ์ด้วย logic เดียวกับ `can_change_status` ใน template (ตรวจว่าอยู่ใน responsible_persons หรือ notify_persons ของ activity)
- **ไฟล์:** `apps/projects/views.py`

### 5. เจ้าหน้าที่ไม่เห็นปุ่ม "เสร็จสิ้น", "บันทึกรายงาน" และ Alert แจ้งเตือน

- **ปัญหา:** ในหน้ารายละเอียดกิจกรรม เจ้าหน้าที่ที่มีสิทธิ์ระดับกิจกรรม (`can_change_status = True`) ไม่เห็น UI ที่ควรจะเห็น
- **สาเหตุ:** Template ใช้เงื่อนไข `can_action` (project-level) แทน `can_change_status` (activity-level)
- **แก้ไข:** เปลี่ยนเงื่อนไขใน template ให้ใช้ `can_change_status` สำหรับ:
  - ปุ่ม "เสร็จสิ้นกิจกรรม / เปิดใหม่"
  - ปุ่ม "บันทึกรายงาน" (header + empty state)
  - Alert แจ้งเตือน (unlinked expense, ยังไม่มีแหล่งเงิน ฯลฯ)
- **ไฟล์:** `templates/projects/activity_detail.html`

### 6. Modal ยืนยันก่อนกดปุ่ม "เสร็จสิ้นกิจกรรม"

- **เหตุผล:** ต้องการให้ผู้ใช้ตรวจสอบสถานะก่อนสิ้นสุดกิจกรรม โดยเฉพาะกรณีมี alert ค้างอยู่
- **การดำเนินการ:**
  - **กรณีไม่มี alert:** Modal แสดงข้อความ "ดำเนินการครบถ้วน" พร้อม checklist สีเขียว กดยืนยันได้ทันที
  - **กรณีมี alert:** Modal แสดงรายการ alert สีส้ม บังคับใส่ comment อธิบายเหตุผล ก่อนกดยืนยัน
  - บันทึก comment ไว้ใน Audit Log (ไม่ต้องเพิ่ม field ใน DB)
  - JavaScript อ่านค่า alert จาก Django template variables โดยตรง
- **ไฟล์:** `templates/projects/activity_detail.html`, `apps/projects/views.py`

### 7. ตรวจสอบ RBAC ทั้งระบบ (Full Review) และแก้ไข 2 จุดสุดท้าย

- **การดำเนินการ:** ตรวจสอบ views, templates, utils ทั้งหมดในทุก app
- **พบและแก้ไขเพิ่มเติม 2 จุด:**

  **7.1 ปุ่มแก้ไข/ลบรายงานกิจกรรม แสดงให้ทุก role เห็น**
  - ปัญหา: ลิงก์ "แก้ไข" / "ลบ" รายงานกิจกรรม ไม่มีเงื่อนไข role ทำให้ผู้ที่ดูอย่างเดียวเห็นปุ่มและได้ 403 เมื่อกด
  - แก้ไข: wrap ด้วย `{% if can_change_status %}`
  - ไฟล์: `templates/projects/activity_detail.html`

  **7.2 ปุ่ม "บันทึกเบิกจ่าย" แสดงให้ executive เห็น**
  - ปัญหา: executive เป็น read-only แต่เห็นปุ่มสร้าง expense ทั้งใน header และ empty state
  - แก้ไข: wrap ด้วย `{% if user.profile.role != "executive" %}`
  - ไฟล์: `templates/budget/expense_list.html`

### 8. ปรับ Sidebar — Collapsible Admin Section + Sub-items

- **ปัญหา:** Admin เห็นเมนูรวม 17 รายการ จอ laptop 768px ต้อง scroll (รวมสูง ~784px)
- **วิเคราะห์:** สาเหตุหลักคือ Admin section 8 รายการที่แสดงแบบ flat ตลอดเวลา
- **การดำเนินการ:**
  - **Collapsible Admin Section:** ห่อ 8 รายการใน section "จัดการระบบ" ด้วยปุ่ม toggle + animation (max-height transition)
  - **Auto-expand:** ถ้า URL ปัจจุบันอยู่ในหน้า admin จะเปิด section อัตโนมัติ
  - **localStorage:** จำสถานะเปิด/ปิดข้ามหน้า
  - **Sub-items:** `ปฏิทินโครงการ` indent ใต้ `โครงการ` และ `อนุมัติรายการ` indent ใต้ `งบประมาณ` ด้วย border-left accent
  - **Compact spacing:** ลด `py-2.5` → `py-2` ทุกรายการ ประหยัด ~90px
  - **แก้ bug:** `แดชบอร์ด` active state เดิม match ทุกหน้าใน dashboard app แก้เป็น url_name == 'index' เท่านั้น
- **ผล:** Admin section ปิด = 424px, เปิด = 680px (จากเดิม 784px ทุกเวลา)
- **ไฟล์:** `templates/components/sidebar.html`

### 9. รวมเมนูผู้ใช้งาน / รออนุมัติ / Auto-Approve เป็น Tabbed Page

- **เหตุผล:** 3 เมนูมีความเกี่ยวข้องกัน (ล้วนเป็นการจัดการผู้ใช้) และอยู่ใน Admin section ที่ต้องการประหยัดพื้นที่
- **การดำเนินการ:**
  - เพิ่ม **Tab Bar** (ผู้ใช้งานทั้งหมด / รออนุมัติ [badge] / Auto-Approve) ในทั้ง 3 template
  - ไม่เปลี่ยน URL หรือ backend views เลย — tab แต่ละอันลิงก์ไป URL เดิม
  - Sidebar เปลี่ยน 3 รายการ → 1 รายการ "ผู้ใช้งาน" พร้อม badge แสดงจำนวนรออนุมัติ
  - Active state ของ sidebar item ครอบทุก URL ที่เกี่ยวข้อง (user_list, pending_user_list, approved_org_list ฯลฯ)
- **ผล:** Admin section ลดจาก 8 → 6 รายการ (ประหยัด 2 แถว)
- **ไฟล์:** `templates/manage/user_list.html`, `templates/manage/pending_user_list.html`, `templates/manage/approved_org_list.html`, `templates/components/sidebar.html`

## สรุปตรรกะ RBAC ที่ใช้ทั่วระบบ (หลังแก้ไข)

| เงื่อนไข | ความหมาย | ใช้กับ |
|----------|----------|--------|
| `can_change_status` | planner/head/admin **หรือ** อยู่ใน responsible_persons/notify_persons ของ **กิจกรรม** | ปุ่ม เสร็จสิ้น, บันทึกรายงาน, แก้ไข/ลบรายงาน, alert |
| `can_action` | Actionable ระดับ **โครงการ** | สร้าง expense, โอนงบ |
| `role == "executive"` | อ่านอย่างเดียว ทุกแผนก | ซ่อนปุ่ม write ทั้งหมด |
| `get_viewable_projects()` | โครงการที่มีสิทธิ์ **ดู** | Dashboard, Timeline, Reports |
| `get_actionable_projects()` | โครงการที่มีสิทธิ์ **แก้ไข** | CRUD operations |
